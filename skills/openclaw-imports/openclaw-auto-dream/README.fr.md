### Pourquoi MyClaw Change Tout

La plupart des outils IA vous donnent une fenêtre de chat. MyClaw vous donne un **serveur**.

Chaque instance MyClaw exécute [OpenClaw](https://github.com/openclaw/openclaw) — le runtime d'agent IA open-source — sur une infrastructure dédiée. Votre agent dispose de :

- 🖥️ **Un serveur Linux complet** — pas un bac à sable, pas un conteneur, une vraie machine
- 🌐 **Internet sans restriction** — naviguer, scraper, appeler des APIs, envoyer des messages
- ⏰ **Cron et tâches en arrière-plan** — votre agent travaille pendant que vous dormez
- 📁 **Système de fichiers persistant** — workspace, mémoire, configs, tout à vous
- 🧩 **Écosystème de skills** — installez de nouvelles capacités depuis [ClawHub](https://myclaw.ai/skills) en une commande
- 🔌 **Intégrations de canaux** — Telegram, Discord, Slack, WhatsApp, et plus
- 🔒 **Vos données, votre serveur** — pas d'infrastructure partagée, pas de data mining

C'est pourquoi Auto-Dream existe. Parce que quand votre agent IA a un serveur persistant, des fichiers persistants et des relations persistantes avec vous — **la mémoire n'est pas un bonus. C'est le fondement de l'intelligence.**

## Le Problème

Chaque agent IA oublie. La session se termine, le contexte disparaît. Les fichiers s'accumulent. Quelle était cette décision d'il y a deux semaines ? Quel workflow a fonctionné la dernière fois ? Votre agent souffre d'amnésie — fonctionnel, mais oubliant tout dès qu'il s'endort.

**Auto-Dream corrige cela.** Comme le cerveau humain consolide ses souvenirs pendant le sommeil, Auto-Dream exécute des « cycles de rêve » périodiques qui scannent, extraient, organisent, notent, relient et élaguent les connaissances de votre agent — automatiquement, en toute sécurité et intelligemment.

### Pourquoi C'est Important pour les Utilisateurs MyClaw

Votre agent MyClaw fonctionne 24h/24 7j/7. Il gère vos projets, suit vos décisions, se souvient de vos préférences, gère vos workflows. Au fil des semaines et des mois, il accumule des milliers d'entrées de logs quotidiens. Sans Auto-Dream, ces connaissances restent dans des fichiers bruts — non triées, non notées, déconnectées. Votre agent a les données mais ne peut pas *réfléchir* dessus.

Auto-Dream transforme votre agent d'un outil qui exécute des commandes en un partenaire qui **comprend le contexte, apprend de l'historique et fait des connexions que vous n'aviez pas vues**.

### Cinq Couches de Mémoire

| Couche | Stockage | Contenu |
|--------|----------|---------|
| **Travail** | Plugin LCM (optionnel, détecté à l'installation) | Compression de contexte en temps réel et rappel sémantique |
| **Épisodique** | `memory/episodes/*.md` | Récits de projets, chronologies d'événements, arcs narratifs |
| **Long terme** | `MEMORY.md` | Faits, décisions, personnes, jalons, stratégie |
| **Procédurale** | `memory/procedures.md` | Workflows, préférences, patterns d'outils, raccourcis |
| **Index** | `memory/index.json` | Métadonnées, scores d'importance, relations, stats de santé |

## Fonctionnalités

### 🧠 Cycle de Rêve Cognitif

S'exécute automatiquement via cron (par défaut : 4h du matin quotidiennement). Trois phases :

1. **Collecter** — Scanne les logs quotidiens non consolidés (7 derniers jours), détecte les marqueurs de priorité (`⚠️ PERMANENT`, `🔥 HIGH`, `📌 PIN`, `<!-- important -->`), extrait les décisions / personnes / faits / projets / leçons / procédures / fils ouverts

2. **Consolider** — Route chaque insight vers la couche mémoire appropriée, effectue la déduplication sémantique, attribue des identifiants uniques (`mem_NNN`), crée des liens de relation entre les entrées connectées

3. **Évaluer** — Score l'importance avec `base_weight × recency × reference_boost / 8.0`, applique les courbes d'oubli (>90 jours + faible importance → archivé, jamais supprimé), calcule le score de santé à 5 métriques, génère 1 à 3 observations non évidentes, rédige le rapport de rêve, envoie la notification

### 📊 Scoring d'Importance

Chaque entrée reçoit un score à chaque cycle de rêve :

```
importance = (base_weight × recency_factor × reference_boost) / 8.0
```

- **Récence** : `max(0.1, 1.0 - days/180)` — décroissance graduelle sur 6 mois
- **Références** : `log₂(count + 1)` — boost logarithmique pour les entrées fréquemment référencées
- **Marqueurs** : `🔥 HIGH` double le poids de base ; `⚠️ PERMANENT` score toujours 1.0

### 📉 Oubli Intelligent

Les souvenirs ne sont pas supprimés — ils sont gracieusement archivés :
- L'entrée doit être >90 jours sans référence ET importance < 0.3
- Compressée en un résumé d'une ligne dans `memory/archive.md`
- L'identifiant original est préservé pour le suivi des relations
- Les entrées `⚠️ PERMANENT` et `📌 PIN` sont immunisées

### 🕸️ Graphe de Connaissances

Les entrées sont liées par des relations sémantiques. La métrique d'accessibilité mesure la connectivité du graphe :
- Algorithme union-find sur toutes les relations d'entrées
- Détecte les clusters de connaissances isolés
- Suggère des références croisées pour améliorer la cohérence

### 🩺 Score de Santé (5 Métriques)

```
health = (freshness×0.25 + coverage×0.25 + coherence×0.2 + efficiency×0.15 + reachability×0.15) × 100
```

| Métrique | Ce qu'elle mesure |
|----------|-------------------|
| **Fraîcheur** | % d'entrées référencées dans les 30 derniers jours |
| **Couverture** | % de catégories de connaissances mises à jour dans les 14 derniers jours |
| **Cohérence** | % d'entrées avec au moins un lien de relation |
| **Efficacité** | Concision de MEMORY.md (inversement proportionnel au nombre de lignes) |
| **Accessibilité** | Qualité de la connectivité du graphe mémoire |

### 🔔 Notifications Push

Les résultats de rêve sont livrés automatiquement dans votre chat :

| Niveau | Ce que vous recevez |
|--------|---------------------|
| `silent` | Rien — journalisé dans dream-log.md uniquement |
| `summary` | `🌀 Santé : 82/100 \| +5 nouveaux, ~3 mis à jour, -1 archivé \| 💡 Insight principal` |
| `full` | Rapport de rêve complet avec toutes les sections |

Un tableau de bord HTML sans dépendances avec :
- Jauge de santé animée (0–100)
- 5 cartes de métriques avec codes couleur
- Graphique en anneau de distribution mémoire
- Histogramme d'importance
- Graphique linéaire de tendance de santé (30 derniers cycles)
- Visualisation du graphe de connaissances en force dirigée
- Changements récents, insights, suggestions, entrées périmées

Pour le générer : *« Afficher le tableau de bord mémoire »*

### 🔄 Migration Inter-Instances

Déplacez des souvenirs entre instances MyClaw :

```
"Export memory bundle"    →  memory/export-2026-03-28.json
"Import memory bundle"    →  fusionne avec l'existant (le plus récent gagne)
"Export only procedures"  →  export sélectif par couche
```

Format de bundle JSON portable avec métadonnées complètes, résolution de conflits et sauvegarde pré-import. Parfait pour les utilisateurs gérant plusieurs instances MyClaw ou migrant vers un nouveau serveur.

### 🔮 Insights de Rêve

Après chaque cycle, 1 à 3 observations non évidentes :
- **Connexions de patterns** — *« La stratégie du Projet X reflète ce qui a fonctionné pour le Projet Y »*
- **Tendances temporelles** — *« Les décisions stratégiques se regroupent le lundi — planification hebdomadaire détectée »*
- **Détection de lacunes** — *« Aucune leçon enregistrée pour les 4 derniers projets — rétrospectives en retard »*
- **Alertes de tendance** — *« Santé en déclin sur 3 cycles : 85→79→72 — entrées périmées en accumulation »*
- **Analyse de densité** — *« mem_042 référencé par 8 entrées mais n'a aucun lien sortant »*

## Sécurité

| Règle | Pourquoi |
|-------|----------|
| Ne jamais supprimer les logs quotidiens | Source de vérité immuable |
| Ne jamais retirer `⚠️ PERMANENT` | La protection utilisateur est absolue |
| Les épisodes sont en ajout seul | L'historique narratif est préservé à jamais |
| Sauvegarde auto sur >30% de changements | Empêche la corruption accidentelle |
| Sauvegarde de l'index à chaque cycle | Toujours récupérable |
| Politique de secrets | Ne consolide que les secrets déjà présents |

## Fonctionnement Interne

Auto-Dream utilise les primitives natives d'OpenClaw — la même infrastructure qui propulse chaque instance MyClaw :

| Primitive | Rôle |
|-----------|------|
| **Cron** | Planifie les cycles de rêve à des intervalles configurables |
| **Sessions isolées** | Exécute la consolidation sans polluer l'historique du chat principal |
| **Système de fichiers** | Lit/écrit les fichiers mémoire à travers les cinq couches |
| **LCM** | Fournit la compression de la mémoire de travail pour les longues conversations |

Aucune dépendance externe. Aucune clé API. Aucune base de données. Juste des fichiers et de l'intelligence.
