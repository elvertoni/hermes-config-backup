## Проблема

Каждый ИИ-агент забывает. Сессия завершается — контекст исчезает. Файлы накапливаются. Какое решение было принято две недели назад? Какой рабочий процесс сработал в прошлый раз? У вашего агента амнезия — он функционален, но забывает всё в тот момент, когда засыпает.

**Auto-Dream решает эту проблему.** Подобно тому, как человеческий мозг консолидирует воспоминания во время сна, Auto-Dream запускает периодические «циклы сновидений», которые сканируют, извлекают, организуют, оценивают, связывают и очищают знания вашего агента — автоматически, безопасно и интеллектуально.

### Почему Это Важно для Пользователей MyClaw

Ваш агент MyClaw работает 24/7. Он управляет вашими проектами, отслеживает ваши решения, помнит ваши предпочтения, управляет вашими рабочими процессами. Неделя за неделей, месяц за месяцем он накапливает тысячи ежедневных записей в логах. Без Auto-Dream эти знания лежат в необработанных файлах — несортированные, неоценённые, несвязанные. У вашего агента есть данные, но он не может *думать* о них.

Auto-Dream превращает вашего агента из инструмента, выполняющего команды, в партнёра, который **понимает контекст, учится на истории и находит связи, которые вы не заметили**.

## Возможности

### 🧠 Когнитивный Цикл Сновидений

Запускается автоматически через cron (по умолчанию: ежедневно в 4 утра). Три фазы:

1. **Сбор** — Сканирует неконсолидированные дневные логи (последние 7 дней), обнаруживает маркеры приоритета (`⚠️ PERMANENT`, `🔥 HIGH`, `📌 PIN`, `<!-- important -->`), извлекает решения / людей / факты / проекты / уроки / процедуры / открытые темы

2. **Консолидация** — Направляет каждый инсайт на правильный уровень памяти, выполняет семантическую дедупликацию, назначает уникальные идентификаторы (`mem_NNN`), создаёт связи между соединёнными записями

3. **Оценка** — Оценивает важность по формуле `base_weight × recency × reference_boost / 8.0`, применяет кривые забывания (>90 дней + низкая важность → архивируется, никогда не удаляется), рассчитывает показатель здоровья по 5 метрикам, генерирует 1–3 неочевидных наблюдения, составляет отчёт о сновидении, отправляет уведомление

### 📊 Оценка Важности

Каждая запись получает оценку при каждом цикле сновидений:

```
importance = (base_weight × recency_factor × reference_boost) / 8.0
```

- **Актуальность**: `max(0.1, 1.0 - days/180)` — постепенное угасание за 6 месяцев
- **Ссылки**: `log₂(count + 1)` — логарифмический бонус для часто упоминаемых записей
- **Маркеры**: `🔥 HIGH` удваивает базовый вес; `⚠️ PERMANENT` всегда даёт оценку 1.0

### 📉 Интеллектуальное Забывание

Воспоминания не удаляются — они изящно архивируются:
- Запись должна быть >90 дней без ссылок И важность < 0.3
- Сжимается до однострочного резюме в `memory/archive.md`
- Оригинальный идентификатор сохраняется для отслеживания связей
- Записи `⚠️ PERMANENT` и `📌 PIN` защищены

### 🕸️ Граф Знаний

Записи связаны семантическими отношениями. Метрика достижимости измеряет связность графа:
- Алгоритм объединения-поиска по всем связям записей
- Обнаруживает изолированные кластеры знаний
- Предлагает перекрёстные ссылки для улучшения когерентности

### 🩺 Показатель Здоровья (5 Метрик)

```
health = (freshness×0.25 + coverage×0.25 + coherence×0.2 + efficiency×0.15 + reachability×0.15) × 100
```

| Метрика | Что измеряет |
|---------|-------------|
| **Свежесть** | % записей, упомянутых за последние 30 дней |
| **Покрытие** | % категорий знаний, обновлённых за последние 14 дней |
| **Когерентность** | % записей с хотя бы одной связью |
| **Эффективность** | Насколько компактным остаётся MEMORY.md (обратно пропорционально количеству строк) |
| **Достижимость** | Насколько хорошо связан граф памяти |

### 🔔 Push-Уведомления

Результаты сновидений доставляются в ваш чат автоматически:

| Уровень | Что вы получаете |
|---------|-----------------|
| `silent` | Ничего — записывается только в dream-log.md |
| `summary` | `🌀 Здоровье: 82/100 \| +5 новых, ~3 обновлено, -1 архивировано \| 💡 Главный инсайт` |
| `full` | Полный отчёт о сновидении со всеми разделами |

### 🔄 Миграция Между Инстансами

Перемещайте воспоминания между инстансами MyClaw:

```
"Export memory bundle"    →  memory/export-2026-03-28.json
"Import memory bundle"    →  слияние с существующими (более новые побеждают)
"Export only procedures"  →  выборочный экспорт по уровням
```

Портативный формат JSON-бандла с полными метаданными, разрешением конфликтов и резервным копированием перед импортом. Идеально для пользователей, управляющих несколькими инстансами MyClaw или переезжающих на новый сервер.

### 🔮 Инсайты Сновидений

После каждого цикла 1–3 неочевидных наблюдения:
- **Связи паттернов** — *«Стратегия Проекта X отражает то, что сработало для Проекта Y»*
- **Временные тренды** — *«Стратегические решения группируются по понедельникам — обнаружено еженедельное планирование»*
- **Обнаружение пробелов** — *«Нет уроков за последние 4 проекта — ретроспективы просрочены»*
- **Предупреждения о трендах** — *«Здоровье снижается 3 цикла: 85→79→72 — накапливаются устаревшие записи»*
- **Анализ плотности** — *«mem_042 упоминается 8 записями, но не имеет исходящих связей»*

## Безопасность

| Правило | Почему |
|---------|--------|
| Никогда не удалять дневные логи | Неизменяемый источник истины |
| Никогда не удалять `⚠️ PERMANENT` | Защита пользователя абсолютна |
| Эпизоды только для добавления | Нарративная история сохраняется навсегда |
| Авто-бэкап при >30% изменений | Предотвращает случайное повреждение |
| Бэкап индекса каждый цикл | Всегда восстановим |
| Политика секретов | Консолидирует только уже существующие секреты |

## Как Это Работает Под Капотом

Auto-Dream использует встроенные примитивы OpenClaw — ту же инфраструктуру, которая обеспечивает работу каждого инстанса MyClaw:

| Примитив | Роль |
|----------|------|
| **Cron** | Планирует циклы сновидений с настраиваемыми интервалами |
| **Изолированные сессии** | Выполняет консолидацию без загрязнения истории основного чата |
| **Файловая система** | Читает/записывает файлы памяти на всех пяти уровнях |
| **LCM** | Обеспечивает сжатие рабочей памяти для длинных разговоров |

Никаких внешних зависимостей. Никаких API-ключей. Никаких баз данных. Только файлы и интеллект.
