## CaracterГӯsticas

### рҹ§  Ciclo de SueГұo Cognitivo

Se ejecuta automГЎticamente vГӯa cron (por defecto: diariamente a las 4 AM). Tres fases:

1. **Recopilar** вҖ” Escanea los logs diarios no consolidados (Гәltimos 7 dГӯas), detecta marcadores de prioridad (`вҡ пёҸ PERMANENT`, `рҹ”Ҙ HIGH`, `рҹ“Ң PIN`, `<!-- important -->`), extrae decisiones / personas / hechos / proyectos / lecciones / procedimientos / hilos abiertos

2. **Consolidar** вҖ” Enruta cada insight a la capa de memoria correcta, realiza deduplicaciГіn semГЎntica, asigna IDs Гәnicos (`mem_NNN`), crea vГӯnculos de relaciГіn entre entradas conectadas

3. **Evaluar** вҖ” PuntГәa la importancia con `base_weight Г— recency Г— reference_boost / 8.0`, aplica curvas de olvido (>90 dГӯas + baja importancia вҶ’ archivado, nunca eliminado), calcula la puntuaciГіn de salud de 5 mГ©tricas, genera 1вҖ“3 observaciones no obvias, redacta el informe del sueГұo, envГӯa la notificaciГіn

### рҹ“Ҡ PuntuaciГіn de Importancia

Cada entrada recibe una puntuaciГіn en cada ciclo de sueГұo:

```

```

- **Recencia**: `max(0.1, 1.0 - days/180)` вҖ” decaimiento gradual en 6 meses
- **Referencias**: `logвӮӮ(count + 1)` вҖ” impulso logarГӯtmico para entradas frecuentemente referenciadas
- **Marcadores**: `рҹ”Ҙ HIGH` duplica el peso base; `вҡ пёҸ PERMANENT` siempre puntГәa 1.0

### рҹ“ү Olvido Inteligente

Los recuerdos no se eliminan вҖ” se archivan con elegancia:
- La entrada debe tener >90 dГӯas sin referencias Y importancia < 0.3
- Se comprime a un resumen de una lГӯnea en `memory/archive.md`
- El ID original se preserva para el seguimiento de relaciones
- Las entradas `вҡ пёҸ PERMANENT` y `рҹ“Ң PIN` son inmunes

Las entradas estГЎn vinculadas por relaciones semГЎnticas. La mГ©trica de alcanzabilidad mide la conectividad del grafo:
